Что это?

Это система, которая помогает правильно работать с кодом в команде.

Как работает?

Позволяет всем членам команды вносить изменения в основную ветку, при этом каждое изменение сохраняются истории .

Зачем это нужно?

Это помогает отслеживать, кто и что изменил в проекте, защищает от случайных ошибок, и позволяет легко возвращаться к старым версиям кода.

Что такое git rebase?

Оно позволяет переписать историю ветки придав нужный вид и позволяет объединить изменения двух веток.

Отличия от git merge

git rebase создает новый набор коммитов, которые применяются поверх целевой ветки, а git merge создает новый коммит слияния, который объединяет изменения из обеих ветвей.

Что такое git fetch

Оно используется для извлечения всех изменений из удаленного репозитория в локальный репозиторий. Он никак не вносит изменения в рабочий каталог. Хранит все изменения в отдельной ветке, называемой веткой удаленного отслеживания.

Отличия от git pull

Команда git fetch копирует изменения только в твой локальный репозиторий. Git pull копирует изменения из удаленного репозитория в рабочий каталог, а git fetch этого не делает. Команда git pull делает и то, и другое.
